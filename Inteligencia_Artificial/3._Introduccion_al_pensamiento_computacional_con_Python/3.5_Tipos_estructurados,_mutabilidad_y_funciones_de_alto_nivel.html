3.5 Tipos estructurados, mutabilidad y funciones de alto nivel

Funciones como objetos

Una de las características más poderosas de Python es que todo es un objeto, incluyendo las funciones. Las funciones en Python son “ciudadanos de primera clase”.

Esto, en sentido amplio, significa que en Python las funciones:

  • Tienen un tipo
  • Se pueden pasar como argumentos de otras funciones
  • Se pueden utilizar en expresiones
  • Se pueden incluir en varias estructuras de datos (como listas, tuplas, diccionarios, etc.)

Funciones en expresiones

Una forma de definir una función en una expresión es utilizando el keyword lambda. lambda tiene la siguiente sintaxis: lambda <vars>: <expresion>.
Ejemplo: sumar = lambda x, y: x + y

Funciones en estructuras de datos

Las funciones también se pueden incluir en diversas estructuras que las permiten almacenar. Por ejemplo, una lista puede guardar diversas funciones a aplicar o un diccionario las puede almacenar como valores.

Tuplas

  • Son secuencias inmutables de datos(no se pueden modificar sus elementos)
  • A diferencia de las cadenas pueden contener cualquier tipo de objetos
  • Pueden utilizarse para devolver varios valores en una funcion
  • Las tuplas se declaran con parentesis
Creacion de tupla vacia my_tuple = ()
Acceso a valores de una tupla my_tuple [0]
Crear tupla con un unico valor numerico my_tuple = (1,) -> Si solo se pone entre parentesis el numero lo tomara como entero, es por ello que se pone la coma.
Desempaquetado de tupla
my_tuple = (1,2,3)
x, y, z = my_tuple
Suma de tuplas (concatenacion de tuplas)
my_tuple = (1,)
other_tuple = (2,3,4)
new_tuple = my_tuple + other_tuple -> Resultado (1,2,3,4)


Rangos

  • Representan una secuencia de enteros
  • Sintaxis: range(comienzo,fin, pasos)
  • Al igula que las cadenas y las tupla, los rangos son inmutables
  • Muy eficientes en uso de memoria
  • Los rango no son inclusivos en el valor de fin, es decir el ultimo numero no lo toma ej: un rango de 0 a 6, llegaria de 0 a 5. -> range(0,6)

Nota: Cuando queremos hacer un value equality se usa el operador == pero cuando queremos hacer object equality se usa el operador is

Listas y mutabilidad

  • Son secuencias de objetos, pero a diferencia de las tuplas, si son mutables, es decir podemos modificar sus valores a traves de los index
  • Cuando modificas una lista, pueden existir efectos efectos secundarios (side effects)
  • Es posible iterar con ellas
  • Hay varios metodos que ayudan a manipular la vista

OJO=> Uno de los efectos secundarios mas comunes en python es que si tenemos una lista a= [1,2] y luego asignamos esas lista a una variable b, al modificar la lista a se nos estaria modificando la lista b. Para solucionar este problema se debe de usar clonacion.

Clonacion
Ejemplos: a = [1,2,3]
Clonando a en b => b = list(a)
Clonando a en c => c = a[::]

List comprenhesion

  • Es una forma concisa de aplicar operaciones a los valores de una secuencia
  • Tambien se pueden aplicar condiciones para filtrar.
  • SINTAXIS -> [ expression for item in list if conditional ]
Ejemplos:
y = [ i for i in range(0,100)]
z = [ i for i in list if i % 2 == 0] -> Saca solo los numeros pares de una lista y la asigna en otra


Diccionarios

  • Son como listas, pero en lugar de usar indices utilizan llaves
  • No tienen orden interno
  • Son mutables
  • Son iterables
  • SINTAXIS -> dic = {llave : valor, llave:valor}

  • Recuperar datos dic['llave'] -> Esto daria un error si la llave no existe
dic.get('llave)
  • Recorrer por llaves for llaves in dic.keys()
  • Recorrer por valores for value in dic.values()
  • Recorrer por llave y valor for llave, valor in dic.items()
  • Buscar una llave dentro de un diccionario => 'llave' in dic
